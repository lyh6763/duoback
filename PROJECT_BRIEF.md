# DUOBACK 프로젝트 기획서 (포트폴리오용)

## 1. 프로젝트 개요
- 프로젝트명: DUOBACK 리디자인
- 유형: 개인 프로젝트
- 기간: 2026.01 (약 2주)
- 역할: 기획, 디자인, 프론트엔드 개발
- 산출물: 6개 HTML 페이지 + 공통 JS/CSS + 문서화

## 2. 배경/문제 정의
- 기존 사이트의 정보 과밀과 시각적 계층 구조 부재
- 전형적인 쇼핑몰 레이아웃으로 브랜드 프리미엄 이미지 약화

## 3. 목표
- 미니멀하고 깔끔한 UI로 브랜드 아이덴티티 강화
- 모바일 우선 반응형 설계
- 직관적인 제품 탐색 경험 제공
- 접근성 기준 준수 (ARIA, 키보드 네비게이션)

## 4. 타겟
- 재택근무 직장인
- 학생
- 인체공학 의자에 관심 있는 사용자

## 5. 핵심 컨셉/키워드
- 프리미엄/미니멀/클린
- 정보 구조 정돈 + 탐색성 강화

## 6. 정보 구조/페이지 구성
- 메인: 텍스트 중심 Hero, 카테고리, 베스트셀러, 브랜드 스토리
- 제품 목록: 그리드/리스트, 정렬, 필터
- 제품 상세: 갤러리, 옵션 선택, 탭, 관련 제품
- 브랜드: 스토리, 핵심 기술, 연혁
- 고객지원: FAQ, A/S 신청, 매장 안내
- 장바구니: 수량 조절, 주문 요약

## 7. 주요 기능/인터랙션
- 제품 데이터 단일 소스(`products-data.js`)
- 실시간 검색(이름/카테고리/키워드)
- 장바구니 상태 관리(LocalStorage)
- IntersectionObserver 기반 스크롤 애니메이션

## 8. 디자인 방향
- 톤: 웜 뉴트럴 + 테크 그린 포인트
- 컬러 예시: #2B7C73(포인트), #B46A4B(보조)
- 타이포: IBM Plex Sans KR, Space Grotesk, Newsreader
- 레이아웃: 텍스트 중심 Hero + 반응형 그리드

## 9. 기술 스택
- HTML5 / CSS3 / Vanilla JS
- CSS Grid, Flexbox, CSS Variables

## 10. 접근성/반응형/최적화
- ARIA 속성, 키보드 네비게이션, ESC 닫기
- Mobile(768) / Tablet(1024) / Desktop(1920)
- Lazy Loading, width/height 지정으로 CLS 방지

## 11. 해결 과제/차별점
- 제품 데이터 중복 제거 → 유지보수성 향상
- 이미지 최적화 → 초기 로딩/CLS 개선
- 접근성 보강 → 키보드/스크린리더 대응
- 모바일 메뉴 시스템 구축
- OG/Twitter 메타태그 추가

## 12. 결과/배운 점
- CSS Grid의 반응형 처리 경험
- IntersectionObserver 활용 경험
- 단일 데이터 소스 관리의 효율성 체감

## 13. 향후 개선
1. 실제 백엔드 API 연동
2. 구조화 데이터(Schema.org) 추가
3. 성능 측정 및 최적화(Lighthouse)
4. 접근성 심화 검증(WCAG 2.1 AA)
